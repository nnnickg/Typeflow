Typeflow Architecture

Interactive map of the Rust engine, C ABI, Swift host, language data, and runtime paths.

Pass-through observer, one replacement side effect.

Typeflow watches physical keys, scores the current token in two layouts, and only mutates host text when a switch decision is strong enough. The app itself inserts the key normally first.

      Workspace modules and ownership boundaries.

      The main split is clean: Rust owns inference and data loading, Swift owns macOS observation and mutation, and the C ABI is the narrow boundary between them.

      What the engine returns

      The output is state notification, not committed text.

      TypeVariants / fieldsMeaning
      InputEvent Letter, Literal, Backspace, EndToken, HostBypass Host-normalized input into the observer state machine.
      Decision Keep, Bypass, Use(Layout) Internal inference result after gates and scoring.
      ObservationAction None, SwitchFutureLayout(Layout), ResetToken Host-visible notification. Only switch actions can trigger replacement and input-source selection.
      ObservationOutput candidates, score, decision, action Rust API shape used by CLI/tests; FFI compresses this to TfObservation plus pending replacement accessors.

      Non-goals baked into the architecture

      These constraints explain several design choices.

      • No inline compositionThe host does not become a text compositor and does not draw overlay text.
      • No per-key host insertionNormal key insertion is owned by the current app. Typeflow only replaces the token after a switch action.
      • No cloud inferenceLanguage models and dictionaries are local artifacts.
      • 34-position physical-key modelCurrent packs are limited to a-z plus punctuation positions ` [ ] ; ' , . \.

      Host policy is separate from inference.

      The Swift host gathers fast base facts on the key path and refreshes Accessibility details off the key path. Rust host-config resolves those facts into flags that the engine understands.

      Normal editor

      No policy block.

      Policy resolution pipeline

      Precedence matters: secure input wins before bundle and terminal checks.

      1

      Secure input

      If IsSecureEventInputEnabled() or the host facts say secure, disable automatic processing and manual switching.

      2

      Terminal bundle

      Known terminal app bundle IDs are full bypass surfaces. Shell input is too risky for automatic mutation.

      3

      Embedded terminal surface

      AX role/identifier/description markers catch terminal panes inside non-terminal apps.

      4

      Configured app blocks

      disable_bundle_ids blocks auto plus manual. disable_auto_bundle_ids blocks automatic switching only.

      Build pipeline

      From corpus inputs to runtime model.

      1

      Corpus + frequency lists

      typeflow-data uses OpenSubtitles and hermitdave FrequencyWords for embedded EN/UK; external specs can use local or HTTP inputs.

      2

      N-gram model

      Builder creates bigram/trigram log probabilities plus smoothed floor values.

      3

      Dictionary FST

      Top words become fst::Map<word, count>. Prefix evidence is precomputed into another artifact.

      4

      Runtime load

      LanguageBundle creates English and secondary LanguagePack instances with keyboard map, n-grams, dictionary, prefix index, and metadata.

      The C ABI is deliberately small.

      Swift sees opaque pointers, fixed-size event/observation structs, config accessors, and explicit pending-replacement buffers. Rust catches panics before they cross the ABI.
